2023-02-20 MIT6.824 ， Distributed Systems ， MapReduce ， Go

Introduction

In this lab I’ll build a MapReduce system: implement a worker process that calls application Map and Reduce functions and handles reading and writing files, and a coordinator process that hands out tasks to workers and copes with failed workers.

My job

Implement a distributed MapReduce by modifying mr/worker.go, mr/master.go, and mr/rpc.go. There will be just one master process, and one or more worker processes executing in parallel.

In this lab you’ll run worker processes all on a single machine. The workers will talk to the master via RPC.

  • fault tolerant:

    The master should notice if a worker hasn’t completed its task in a reasonable amount of time (for this lab, use ten seconds), and give the same task to a different worker.
