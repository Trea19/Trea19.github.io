WebServer 环境配置

2023-01-31 webserver ， ecs ， docker ， ssh

简单记录下环境配置过程：Linux ecs + ssh + docker + vscode远程开发

我选在云服务器进行开发，本地是WSL2 Ubuntu22.04

以下以阿里云服务器Ubuntu Focal 20.04 (LTS)为例，记录环境配置过程。

租云服务器

略

SSH

  1. 在本地linux用ssh root@公网ip连接

  2. 创建用户（此时是root用户，权限太大了，一般在普通用户下进行开发）

    adduser 用户名
usermod -aG sudo 用户名  # 给用户分配sudo权限

  3. 配置免密登录

    exit #退回本地linux
vim ~/.ssh/config  #创建文件

    在文件里输入

    Host myserver
    HostName IP地址或域名
    User 用户名

    退出文件

    创建密钥

    ssh-keygen

    免密登录配置

    ssh-copy-id myserver

    之后就可以用ssh myserver 连接服务器啦

  4. 给毛坯云服务器配环境

    ssh myserver
sudo apt-get update
sudo apt-get install tmux  #装个tmux

Docker

  1. 装docker 官方文档

    建议在tmux里装，找到文档里自己服务器对应的操作系统，复制文档命令即可

    sudo apt-get update
sudo apt-get install \
    ca-certificates \
    curl \
    gnupg \
    lsb-release
    sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
    echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
    sudo apt-get update
    sudo apt-get install docker-ce docker-ce-cli containerd.io docker-compose-plugin
    sudo docker run hello-world

    验证命令 docker --version

  2. docker镜像
    为了避免每次使用docker命令都需要加上sudo权限，将当前用户添加到docker用户组

    sudo usermod -aG docker $USER
newgrp docker

    拉取一个镜像

    docker pull ubuntu:20.04

    查看镜像

    docker images

    *删除镜像

    docker rmi hello-world:latest #之前多余的镜像

  3. docker容器

    利用镜像生成一个容器

    docker [container] create -it ubuntu:20.04

    查看本地的所有容器

    docker ps -a  #所有容器
docker ps #运行中的容器

    启动容器

    docker [container] start CONTAINER

    停止

    docker [container] stop CONTAINER

    重启

    docker [container] restart CONTAINER

    在容器启动状态下，进入容器，类似进入一个虚拟机

    docker [container] attach CONTAINER

    查看所有容器的统计信息，包括CPU、内存、存储、网络等信息

    docker stats
ctrl+c #快捷键退出

    在本地和容器间复制文件

    docker cp xxx CONTAINER:xxx 或 docker cp CONTAINER:xxx xxx

    修改容器限制

    docker update CONTAINER --memory 500MB

    重命名容器

    docker rename CONTAINER1 CONTAINER2

    挂起容器: ctrl + p ,再按 ctrl + q

  4. ssh免密登录docker

    进入docker

    docker attach my_docker_server

    设置密码

    passwd

    安装ssh

    apt-get install ssh
apt-get update
sudo /etc/init.d/ssh start

    ctrl+p ctrl+q挂起容器

    回到本地linux

    仿照前文类似方法，添加用户，配置免密登录 myserver_docker

    即可在本地WSL用ssh myserver_docker 直接进入docker容器里

VSCode远程开发

用云服务器里的vim进行开发时键盘响应有明显延迟，推荐使用VSCode进行远程开发
